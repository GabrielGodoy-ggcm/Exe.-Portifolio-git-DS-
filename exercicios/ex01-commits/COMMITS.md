# Exercício 01 — Commits Semânticos

## Padrão utilizado

`<tipo>(<escopo>): <descrição curta>`

## Tipos usados neste repositório

| Tipo | Significado | Exemplo |
|---|---|---|
| `feat` | Nova funcionalidade ou arquivo | `feat(hello-world): adiciona gerenciador de futebol` |
| `docs` | Alteração em documentação | `docs(readme): atualiza índice` |
| `fix` | Correção de erro | `fix(futebol): resolve conflito entre branches` |
| `chore` | Configuração geral | `chore: inicializa estrutura do repositório` |
| `refactor` | Melhoria sem mudar comportamento | `refactor(futebol): extrai função para arquivo separado` |

## Por que commits semânticos?

Commits semânticos tornam o histórico do repositório legível como uma narrativa.
